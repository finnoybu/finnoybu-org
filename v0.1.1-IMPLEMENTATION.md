## Feature: Snapshot History Engine with 2-Snapshot Retention

All implementation steps completed and validated.

### Implementation Checklist

- ✅ **Filesystem Write Utility**: Atomic writes via temporary file + rename
  - `atomicWriteSnapshot()` - Writes snapshot to `data/snapshots/<domain>/<ISO_UTC>.json`
  - Timestamp format: `YYYY-MM-DDTHH-mm-ssZ` (filesystem-safe)
  - Automatic recovery from write failures

- ✅ **Snapshot Persistence Layer**
  - `persistSnapshot()` - Main persistence entry point
  - `listSnapshotsForDomain()` - Lists snapshots sorted newest first
  - `getLatestSnapshot()` - Query most recent snapshot (with legacy fallback)
  - `getSnapshotHistory()` - Query all snapshots (max 2)

- ✅ **2-Snapshot Retention Policy**
  - `enforceSnapshotRetention()` - Keeps only 2 most recent snapshots
  - Deletes oldest snapshots when count exceeds 2
  - Automatic cleanup after each write

- ✅ **API Endpoints**
  - `GET /api/snapshot/latest?domain=example.com` - Returns most recent snapshot
  - `GET /api/history?domain=example.com` - Returns all snapshots (max 2), newest first
  - Error handling: 400 for missing domain param, 404 for no snapshots found

- ✅ **UI Enhancements**
  - Removed manual "Refresh Snapshot" button
  - Added auto-snapshot on domain dropdown selection
  - Seamless UX: select domain → snapshot created automatically
  - Loading state preserved during async operations

## Storage Model

### Directory Structure
```
data/
├── domains.json                          (legacy store - backwards compatible)
└── snapshots/
    ├── example.com/
    │   ├── 2026-03-02T14-30-45Z.json   (ISO timestamp)
    │   └── 2026-03-02T14-45-30Z.json
    └── finnoybu.com/
        ├── 2026-03-02T15-00-00Z.json
        └── 2026-03-02T15-15-45Z.json
```

### Retention Example
```
Domain added → Snapshot created
                └─ 1 file: 2026-03-02T14-30-45Z.json

Domain selected 5 mins later → Auto-snapshot
                               ├─ 2026-03-02T14-30-45Z.json (older)
                               └─ 2026-03-02T14-35-45Z.json (newer)

Domain selected 5 mins later → Auto-snapshot (retention enforced)
                               ├─ 2026-03-02T14-35-45Z.json (older)
                               └─ 2026-03-02T14-40-45Z.json (newer)
                               [2026-03-02T14-30-45Z.json DELETED]
```
